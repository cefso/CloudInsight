## 二、高优先级问题（影响功能/安全）

### 2.1 后端：无结构化日志
**位置**: 整个后端项目  
**问题**: 全部使用 `print()` 输出错误，生产环境无法做日志级别控制和收集  
**修复**: 引入 `logging` 模块，替换所有 `print(f"...")` 为 `logger.error(...)` / `logger.warning(...)`

### 2.2 后端：响应格式冲突
**位置**: `routers/accounts.py:53`, `routers/inspections.py`  
**问题**: 成功返回 `{"code": 200, "data": ...}`，错误抛 `HTTPException` 返回 `{"detail": "..."}`，前端需处理两种格式  
**修复**: 注册全局异常处理器，统一所有响应为 `{"code": xxx, "message": "...", "data": ...}` 格式

### 2.3 后端：数据库无索引
**位置**: `models/inspection_result.py`, `models/inspection_task.py`  
**问题**: 高频查询字段（`task_id`, `status`, `account_id`）无索引，数据量大时查询变慢  
**修复**: 为以下字段添加索引：
- `inspection_results.task_id`
- `inspection_results.status`
- `inspection_results.account_id`
- `inspection_tasks.status` + `completed_at` 联合索引

### 2.4 后端：加密密钥安全
**位置**: `.env` 文件  
**问题**: `.env` 文件含加密密钥，未加入 `.gitignore`  
**修复**: 将 `.env` 加入 `.gitignore`，移除已提交的密钥

### 2.5 前端：导出功能运行时 Bug
**位置**: `api/inspections.ts:13` + `api/index.ts:8-14`  
**问题**: `exportResults` 使用 `responseType: 'blob'`，但 Axios 拦截器尝试读取 `data.code`，Blob 没有该属性导致永远报错  
**修复**: 拦截器中判断 `responseType`，blob 响应直接返回不走 JSON 解析

### 2.6 前端：无限循环风险
**位置**: `pages/Inspections/Detail/index.tsx:54-69`  
**问题**: `fetchResults` 分页循环无上限保护，API 返回异常时浏览器冻结  
**修复**: 添加最大页数限制（如 50 页）

### 2.7 前端：JSON.parse 无错误处理
**位置**: `pages/Inspections/Detail/index.tsx:94, 156, 180, 202`  
**问题**: `JSON.parse(record.disk_details)` 无 try/catch，格式异常时整个组件崩溃  
**修复**: 包裹 try/catch，异常时返回空数组

## 三、中优先级问题（影响可维护性）

### 3.1 后端：代码重复 - 账号序列化
**位置**: `routers/accounts.py:19-30, 55-64`  
**问题**: 账号字典构造逻辑重复 3 次，`CloudAccountResponse` schema 已定义但未使用  
**修复**: 使用 `CloudAccountResponse.model_validate(account)` 替代手动构造

### 3.2 后端：`__dict__` 直接序列化
**位置**: `routers/inspections.py:63`, `routers/thresholds.py:19`  
**问题**: 暴露 SQLAlchemy 内部属性 `_sa_instance_state`，绕过 Pydantic 验证  
**修复**: 使用已定义的 Pydantic schema 进行序列化

### 3.3 后端：同步阻塞异步框架
**位置**: `services/inspection_engine.py:116`, `services/scheduler.py:57-69`  
**问题**: `time.sleep(0.2)` 同步阻塞；`AsyncIOScheduler` 中执行同步操作阻塞事件循环  
**修复**: 将巡检任务改为独立进程/线程池执行，或使用 `asyncio.sleep`

### 3.4 后端：状态字段无约束
**位置**: `models/inspection_task.py`, `models/inspection_result.py`  
**问题**: `status`、`trigger_type` 使用自由字符串，无枚举约束  
**修复**: 定义 Python Enum，模型使用 `Enum` 类型

### 3.5 后端：`disk_details` 字段语义混乱
**位置**: `models/inspection_result.py:19`  
**问题**: ECS 存磁盘详情，SLB 存监听器+后端服务器，同一字段两种数据结构  
**修复**: 拆分为 `disk_details` 和 `slb_details` 两个字段

### 3.6 前端：类型安全缺失
**位置**: `api/inspections.ts` 全部函数, `api/accounts.ts:15,20,33`  
**问题**: 大量 `any` 类型，`const res: any` 绕过类型检查  
**修复**: 为所有 API 函数添加返回类型注解

### 3.7 前端：硬编码颜色，深色模式失效
**位置**: `pages/Inspections/Detail/index.tsx` 全文  
**问题**: 使用 `#dc2626`, `#16a34a`, `#fef2f2` 等硬编码颜色，不随主题变化  
**修复**: 提取颜色到主题配置，使用 CSS 变量或 Ant Design token

### 3.8 前端：无组件复用
**位置**: 所有页面组件  
**问题**: 页面标题、面包屑、加载状态等模式重复，无共享组件  
**修复**: 提取 `PageHeader`, `StatusTag`, `MetricDisplay` 等通用组件

### 3.9 前端：API 函数组织混乱
**位置**: `api/inspections.ts`, `api/dashboard.ts`  
**问题**: `triggerInspection` 在 dashboard.ts 中，阈值和定时任务 API 在 inspections.ts 中  
**修复**: 按领域拆分：`api/accounts.ts`, `api/inspections.ts`, `api/thresholds.ts`, `api/cron.ts`, `api/dashboard.ts`
