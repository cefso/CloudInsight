## 二、各文件详细内容

### 2.1 核心文件

#### `main.py` — 应用入口
- 配置 logging（格式：`%(asctime)s [%(levelname)s] %(name)s: %(message)s`）
- 定义 lifespan：初始化数据库 + 启动/停止定时调度器
- 注册全局异常处理器：HTTPException 和 Exception 统一返回 `{"code": xxx, "message": "...", "data": null}`
- 配置 CORS 中间件
- 注册 5 个路由模块
- 两个根路由：`/` 和 `/api/health`

#### `config.py` — 配置管理
- 使用 `pydantic-settings` 的 `BaseSettings`
- 配置项：app_name, app_version, debug, database_url, encryption_key, cors_origins
- 从 `.env` 文件加载配置
- `get_settings()` 使用 `@lru_cache` 缓存

#### `database.py` — 数据库连接
- SQLAlchemy 创建 SQLite 引擎（`check_same_thread=False`）
- `SessionLocal` 会话工厂
- `Base` 声明基类
- `get_db()` 依赖注入生成器
- `init_db()` 创建所有表

### 2.5 服务层 (services/)

#### `crypto.py` — 加密服务
- `CryptoService` 类，使用 `cryptography.fernet` 对称加密
- `encrypt(text)` / `decrypt(encrypted_text)` — 加解密
- `mask_ak(ak)` — AK 脱敏（前6后4，中间***）
- 全局单例 `crypto_service`
- 未配置密钥时自动生成并警告

#### `aliyun_client.py` — 阿里云客户端（376行，最大文件）
- `AliyunClient` 类，封装 5 个 SDK 客户端：
  - `CmsClient` — 云监控
  - `SlbClient` — 负载均衡
  - `EcsClient` — 云服务器
  - `RdsClient` — 数据库
  - `BssClient` — 账单（用于到期查询）

**核心方法：**
- `test_connection()` — 测试连接
- `get_metric_data()` — 获取指标（含磁盘过滤逻辑）
- `list_resources()` — 列出资源（分页 + 获取真实名称）
- `_fill_ecs_names()` / `_fill_rds_names()` — 填充实例名称
- `list_slb_instances()` — 列出 SLB 实例
- `get_slb_listeners()` — 获取 SLB 监听器
- `get_slb_health_status()` — 获取 SLB 后端健康状态
- `get_expiring_instances()` — 获取即将到期实例（排除 NotRenewal）

**资源指标配置：**
- ECS: CPUUtilization, memory_usedutilization, diskusage_utilization
- RDS: CpuUsage, MemoryUsage, DiskUsage

**特殊处理：**
- 磁盘指标按 `diskname` 分组，过滤容器挂载点
- ECS/RDS 通过专用 SDK 获取真实实例名称

#### `inspection_engine.py` — 巡检引擎（342行）
- `InspectionEngine` 类

**核心流程：**
1. `run_inspection()` — 主入口
   - 创建/复用任务记录
   - 查询启用的账号
   - 获取告警阈值（默认 90%，警告 = 阈值-10%）
   - 遍历账号执行巡检 + 到期检查
   - 更新任务统计

2. `_inspect_account()` — ECS/RDS/SLB 巡检
   - 遍历地域 + 资源类型
   - SLB 走独立逻辑 `_inspect_slb()`
   - ECS/RDS 逐个资源获取指标
   - 每个指标间隔 0.2s 防限流
   - 三级状态：abnormal > warning > normal

3. `_inspect_slb()` — SLB 巡检
   - 获取实例 → 监听器 → 后端健康
   - 监听器：stopped=警告，其他非running=异常
   - 后端服务器：unavailable=警告，abnormal=异常
   - 实例状态非 active = 异常

4. `_inspect_expiration()` — 到期时间巡检
   - 调用 BssOpenApi 查询 15 天内到期实例
   - <7天=异常，7-15天=警告
   - 存储 product_code, end_time, days_remaining

#### `scheduler.py` — 定时调度
- `TaskScheduler` 类，使用 APScheduler
- `start()` / `stop()` — 生命周期管理
- `add_job()` / `remove_job()` / `update_job()` — CRUD
- `_load_jobs()` — 启动时加载数据库中的定时任务
- `_run_inspection()` — 定时执行巡检

## 六、关键设计模式

1. **统一响应格式** — 所有 API 返回 `{code, message, data}`
2. **加密存储** — AK/SK 使用 Fernet 对称加密
3. **异步巡检** — 手动触发通过 `threading.Thread` 后台执行
4. **三级告警** — normal / warning / abnormal
5. **防限流** — 每个指标请求间隔 0.2s
6. **显式序列化** — 所有路由使用辅助函数序列化，不使用 `__dict__`
